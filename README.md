# SportSim — a chess engine for football

Real match video → SAM 3 temporal masks → symbolic behavioral abstraction →
stochastic-differential player models → CMA-ES evolution → a playable board.

## The idea

Chess engines understand a position because every piece has known dynamics.
SportSim learns the dynamics of real players from video: each player becomes a
7-parameter stochastic differential equation (anchor spring, drag, ball
coupling, noise, top speed, ball-following gains), and evolution tunes those
parameters until the simulated player moves like the filmed one. The fitted
parameters ARE the player's "piece rules" — then the engine can replay, predict
(shadow), extrapolate (dream), score transfers, and search counter-strategies.

## Run

```bash
source .venv/bin/activate
cd pipeline
python 01_segment.py --video ../data/raw/clip.mp4 --repo ../models/sam3
python 02_track.py && python 02b_homography.py && python 03_features.py
python 05_evolve.py --selftest        # pipeline must recover known params first
python 05_evolve.py
python 06_simulate.py && python 07_export.py
python 08_transitions.py --selftest && python 08_transitions.py
python 09_evolve_transitions.py --selftest && python 09_evolve_transitions.py
cd .. && python3 -m http.server -d viz 8777   # open http://localhost:8777
                                              # laboratory: http://localhost:8777/lab.html
```

## The possession graph (layer two)

Layer one learns how players MOVE; layer two learns what they DO with the
ball. Each player is a vector of eleven conditional probability distributions
— pass completion given distance, angle to the attacking direction, pressure,
lane risk and the receiver's control; an execution error cone (angular +
power noise, wider for weak passers and long balls) that decides where failed
passes actually land; carry survival per meter against each tackler; shot
conversion given distance and the angle subtended by the posts, with placement
sampled inside the goal mouth; interception and deflection, everything that
moves the ball. The ball is a token on a stochastic transition graph and the
match is the chain those transitions generate — under the laws of the game: a
ball over the line is dead and comes back as a throw-in, corner or goal kick
by the right team at the right spot, goals restart with kickoffs, the sweeper
keeps goal, saves are parried behind, shots ring off the woodwork.
Probabilities are extracted from just two games containing all players
(synthetic for now — generated by hidden "sealed" vectors θ*; the cloud
segmentation of real footage later emits the same event schema and nothing
downstream changes). A genetic algorithm evolves all 132 genes until simulated
transition statistics match the training games, validated against six held-out
games. The laboratory shows this happening in real time; the theater plays it
in 3D with rapier physics at the goalmouth.

## Honesty ledger (one 18 s clip)

Identifiable: σ (noise), γ (drag), v_max (lower bound), ρ (home base).
Ridge-coupled: k_home↔γ, k_ball↔β — the L2 prior picks the point; restart
spread is the honest uncertainty. Frozen by design: stamina, per-player
separation, pass policies (need minutes-to-hours of footage — that's what the
H100 package is for). Fitness < 1.0 means beating inertial extrapolation.
